## GCP 서버 설정 가이드

이 가이드는 Compute Engine을 이용해 인스턴스를 생성하고, SSH 접속을 설정하는 과정을 포함합니다.

#### 1. Compute Engine 인스턴스 생성

1. GCP 콘솔에 로그인합니다.
2. Compute Engine 메뉴로 이동한 후, "인스턴스 만들기"를 선택합니다.
3. 원하는 설정을 지정합니다:
   - 리전: 원하는 리전을 선택합니다. ~~서울 사람이라 서울로~~
   - 구성: 인스턴스 유형과 기타 설정을 선택합니다.
   - 운영체제: Ubuntu 20.04.6을 선택합니다.
   - 트래픽 허용: HTTP 및 HTTPS 트래픽 허용을 체크합니다.

4. 모든 설정을 완료한 후, "만들기" 버튼을 클릭하여 인스턴스를 생성합니다.

#### 2. 외부 IP 고정 할당

접속해야 하는 IP가 변경되면 곤란하니까 고정시켜 줍니다. 

1. VPC 네트워크 - IP 주소 - 외부 고정 IP 주소 예약
2. 리전 맞춰 설정 후 연결 대상을 아까 생성한 인스턴스로 잡아줍니다. 

#### 3. SSH 키 생성 및 등록

1. 로컬 머신에서 터미널을 엽니다.
2. SSH 키를 생성합니다. 아래 명령어를 사용합니다:

   ```bash
   ssh-keygen -t rsa -b 4096 -C ${ID}
   ```

3. 명령어 실행 후, 두 번 엔터를 눌러 기본 경로에 키를 저장합니다.
4. 공개키(id_rsa.pub)와 개인키(id_rsa)가 생성됩니다. 공개키(.pub)를 열어 내용물을 복사합니다.

5. GCP 콘솔에서 방금 생성한 VM 인스턴스의 세부정보 페이지로 이동합니다.
6. 설정 - 메타 데이터 - SSH 키 항목을 찾습니다.
7. 복사한 공개키 내용을 SSH 키 항목에 등록합니다.

#### 4. SSH 접속을 위한 방화벽 설정

1. 인스턴스의 22번 포트를 열어줍니다. 인스턴스 세부정보 페이지에서 "SSH 연결" 버튼을 눌러 인스턴스에 접속합니다.
2. UFW(방화벽)를 설정합니다:

   ```bash
   sudo ufw status
   > Status: inactive
   ```

   만약 상태가 `inactive`이라면, UFW를 활성화합니다:

   ```bash
   sudo ufw enable
   > Status: active
   ```

   그 후, 22번 포트를 허용합니다:

   ```bash
   sudo ufw allow 22
   > Rule added
   > Rule added (v6)
   ```

#### 5. 연결 테스트

1. 로컬 머신에서 서버를 핑 테스트합니다:

   ```bash
   ping ${외부IP주소}
   > ${외부IP주소}의 응답: 바이트=32 시간=33ms TTL=57
   ```

   인스턴스에서 정상적으로 응답이 오는 것을 확인할 수 있습니다.

2. SSH 접속을 시도합니다. 터미널에서 아래 명령어를 사용합니다:

   ```bash
   ssh -i id_rsa ${ID}@${외부IP주소}
   ```

   접속이 성공하면 다음과 같은 메시지가 표시됩니다:

   ```bash
   > Welcome to Ubuntu 20.04.6 LTS (GNU/Linux 5.15.0-1060-gcp x86_64)
   ```

이제 GCP 서버 생성 완료!
